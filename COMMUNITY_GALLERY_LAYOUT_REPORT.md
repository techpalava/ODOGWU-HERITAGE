# Community Gallery Image-to-Caption Layout Report

## Summary of Refinements
- **New Image-to-Caption Proportions**: Redesigned the gallery cards to strictly enforce an approximate 80/20 ratio. The image area now occupies 75% (mobile), 78% (tablet), and 80% (desktop) of the card's total height, ensuring the actual photograph acts as the true hero element of the component. The caption area elegantly occupies the remaining 25%, 22%, and 20% respectively.
- **Responsive Layout Behaviour**: Leveraged CSS aspect ratios (e.g., `aspect-[3/4]`, `aspect-[4/5]`) on the parent cards across both the Homepage carousel and the Gallery grid. This allows the cards to respond fluently to viewport scaling while maintaining structural consistency, preventing awkwardly stretched or squashed image frames.
- **CSS/Layout Adjustments**: Transitioned from absolute floating text overlays to a strictly separated flex-column layout. The caption area is now an independent block beneath the image (`h-[20-25%] flex flex-col justify-between p-4 sm:p-6`), ensuring captions never obscure the photographs while remaining cleanly aligned with a top-border delimiter. Lengthy captions are gracefully truncated using `line-clamp-2` or `line-clamp-3`.
- **Image Rendering Improvements**: Retained `object-contain` combined with a soft background (`bg-heritage-cream/5`). This means images perfectly scale up to fill their 80% boundary without distorting or clipping details, and any differing aspect ratios are smoothly blended with the background color. 
- **Performance Considerations**: All `motion` layout animations, lazy loading behaviors, and high-performance flex rendering behaviors were preserved. The flex proportioning is CSS-native, removing the need for computationally heavy JavaScript resize listeners.
- **Scope Confirmation**: Confirmed that no components outside of the Community Gallery visual card structures were affected. Firestore queries, data schemas, batch attributes, and routing mechanisms remain completely intact and untouched.
